Background

The ferries in Washington State are equiped with anemometers that report wind observations and the position from the vessels as they move from port to port. This project was a collaboration between University of Washington and Washingotn State Ferries. The project page can be found here.

My Project

I was interested in animating the observations from the the ferries. I wrote a web scraper using Python to gather and store the observations in MySQL. I then wrote a simple web page that includes an embedded google map. The page lets the user select the ferry (by vessel name) and the date. You can then press Play to animate the ferry for that day. The wind observation is represented by an arrow which points in the direction of the observed wind and is sized according to the magnitude of the wind speed. The arrows are rendered using canvas. The browser gets the wind data by calling a URL using Ajax. The URL is part of a small service that I implemented using PHP.

Future work

There are many improvements I'm hoping to make. First of all, I'd like to scap the PHP service I wrote, as it is kind of clunky (I wrote all of this over 5 years ago). I would like to implement a general RESTful API to query the wind data I've been scraping over the years. I would also like to let users specify start & end times for the period in which they want to animate the data. (Currently you have to choose a date and you get 24 hours of data starting and ending at Midnight). I would also like to display a graph of the data that would show the data over the selected time period.
